# Lock In Extension

## Install Extension

If you just want to use the extension, this is all you need.

1. Download this repository as a ZIP and extract it.
2. Open `chrome://extensions/` (or `edge://extensions/`).
3. Enable **Developer mode**.
4. Click **Load unpacked**.
5. Select the `dist` folder.
6. Done. The extension is now installed.

---

Lock In is a Chromium browser extension that helps limit distracting browsing with local-only data storage.

You configure per-site rules:

- Max limit per day (optional)
- Session time limit (optional)
- Open limit per day (optional)
- Reflect delay + custom message
- Mode:
  - `strict`: no bonus time when limit is reached
  - `im a bitch`: allow repeated `Add 5 minutes` actions

It also supports:

- Global extension ON/OFF toggle
- Session-aware reflect flow (reflect appears once per session)
- Session reset when all tabs of the site close OR when user is away from that site beyond a configurable inactivity threshold (default 5 min)

## Privacy

- No backend
- No cloud sync
- No telemetry
- All data is stored in `chrome.storage.local` on the user device

## How Browser Extensions Work (MV3)

This project uses Manifest V3 architecture:

1. Service worker (`src/background/main.ts`)
	- Runs extension logic in the background.
	- Owns persistent state updates and access decisions.
	- Handles message requests from UI/content scripts.

2. Content script (`src/content/main.ts`)
	- Runs inside matching web pages.
	- Requests access decisions from service worker.
	- Renders reflect and block modals directly in page context.
	- Sends heartbeats for active-tab-only time tracking.

3. Popup (`src/popup/*`)
	- Quick controls: global ON/OFF, active tab summary, open settings.

4. Options page (`src/options/*`)
	- Full configuration UI for site rules and advanced settings.

5. Manifest (`manifest.config.ts`)
	- Declares permissions, content scripts, service worker, popup, options page.

## How This Extension Enforces Limits

### 1) Site matching

- Input domain is normalized (example: `www.youtube.com` -> `youtube.com`).
- Matching includes subdomains (example: `m.youtube.com`, `music.youtube.com`).

### 2) Session model

- A domain has one active session shared across its tabs.
- Reflect modal appears once when a session starts.
- Session resets if:
  - all tabs for that domain are closed, or
  - user is inactive/away longer than threshold.

### 3) Access decision order

For each visit/check:

1. Global extension OFF -> allow
2. No matching configured site -> allow
3. Open limit exceeded -> block
4. Daily/session time exceeded -> block
5. Reflect not completed for current session -> reflect modal
6. Otherwise -> allow

### 4) Add 5 minutes flow

- When blocked by a time limit:
  - `strict` mode: only `Leave`
  - `im a bitch` mode: `Add 5 minutes` + `Leave`
- After successful add-5, reflect modal is required again before browsing.

## Developer Setup

### Requirements

- Node.js LTS installed
- Chromium browser (Chrome, Edge, Brave, Opera)

### Install dependencies

```bash
npm install
```

### Build extension

```bash
npm run build
```

### Dev watch mode

```bash
npm run dev
```

### Load unpacked extension (Chromium)

1. Open `chrome://extensions/` (or `edge://extensions/`).
2. Enable Developer mode.
3. Click Load unpacked.
4. Select the `dist` directory generated by build/dev.

## Basic Usage

1. Open extension popup.
2. Click `Open full settings`.
3. Add a domain and configure limits.
4. Choose mode (`strict` or `im a bitch`).
5. Set reflect delay/message.
6. Keep extension ON.
7. Browse tracked site and verify reflect + blocking behavior.

## Permissions Used

- `storage`: persist config/stats locally
- `tabs`: tab/session management and cleanup
- `alarms`: periodic housekeeping
- `activeTab`: current-tab dashboard context
- `host_permissions: <all_urls>`: content script + site enforcement on configured sites

## Project Structure

- `manifest.config.ts`: MV3 manifest definition
- `src/background/main.ts`: core policy engine + storage mutations
- `src/background/storage.ts`: serialized state access wrapper
- `src/content/main.ts`: in-page modal UX + heartbeat client
- `src/options/`: complete site rule management UI
- `src/popup/`: quick controls and active-site summary
- `src/shared/`: shared types and utility functions

## Notes for Development

- The background layer is the source of truth for limit decisions.
- Content script never decides policy on its own.
- Time tracking uses active-page heartbeat to avoid double-counting across tabs.
- Daily counters reset at local midnight.
